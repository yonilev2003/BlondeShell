## FANVUE INTEGRATION (COMPLETE — images only)

### Auth flow
- One-time PKCE auth: `npm run fanvue:auth` → browser → Railway catches callback → saves tokens to Supabase
- Railway route: `GET /fanvue-callback` in webhook/server.js
- Tokens: `fanvue_tokens` table (id='singleton', access_token, refresh_token)
- Scopes: `read:self read:media read:post read:fan read:insights write:media write:post offline_access offline`

### lib/fanvue.js exports
- `uploadMediaFromUrl(publicUrl, name)` — POST /media/uploads with url → PATCH complete → poll GET /media/{uuid} until status="ready" (max 2 min)
  - Note: media status cycles through "error" during processing — ignore, only stop on "ready" or timeout
- `getAccessToken()` — refresh_token grant via Basic Auth, saves new token to Supabase
- `createPost({ mediaUuids, caption, isFree, audience })` — POST /posts
- `scheduleFanvuePost({ mediaUuids, caption, scheduledAt, isFree, audience })`

### Confirmed Fanvue API field names
- Caption field: `text` (not content, description, body, caption)
- Media field: `mediaUuids` (camelCase array)
- Audience: `"followers-and-subscribers"` or `"subscribers"`
- isFree: `isFree` (camelCase)
- Token exchange: Basic Auth header `Authorization: Basic base64(clientId:clientSecret)`
- Video URL-import: PATCH returns 500 — Fanvue doesn't support video URL-import
- Video binary upload: no signed URL endpoint found — skip videos on Fanvue for now

### Marketing agent Fanvue integration
- After Publer scheduling, fires Fanvue posts async (non-blocking Promise.all, fire-and-forget)
- T1 images → audience="followers-and-subscribers", isFree=true
- T2 images → audience="subscribers", isFree=false
- Videos → skipped (Fanvue API limitation)
